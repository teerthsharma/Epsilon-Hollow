## 2024-05-19 - Improved ML Tensor Matrix Multiplication Cache Locality
**Learning:** In Rust numerical code like the `matmul` function within `aether-core`, standard `for i in 0..m { for j in 0..n { for l in 0..k { ... } } }` looping over strides is prone to serious cache locality degradation because writing to `data_c` inside the innermost loop with a varying `j` continuously sweeps through memory cachelines, causing severe cache misses.
**Action:** Always swap matrix multiplication loops from `i-j-l` to `i-l-j` or block the loops when possible. The `i-l-j` order allows accumulating along rows by varying `j` in the innermost loop (where memory is sequentially arranged and read once per `i-l` block), improving locality. In tests this change yielded an almost 4x speedup on this codebase.
