## 2024-05-18 - Next.js React Hook Purity & Module-Level Optimization
**Learning:** Next.js enforces strict React hook purity via ESLint (`react-hooks/purity`). Impure functions like `Math.random()` are strictly forbidden during the render phase or inside standard hooks like `useMemo` because they produce unstable results leading to unpredictable re-renders.
**Action:** When initializing large graphical/mathematical datasets that use impure functions (like `Math.random()`), hoist the generation logic out of the component entirely into module-level scope (e.g. an IIFE). This executes the O(N) generation exactly once upon module initialization, providing a significant one-time computation performance win (zero cost on component mount/re-renders) while perfectly resolving the hook purity constraints.
