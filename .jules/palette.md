## 2024-06-11 - Accessibility of dynamic aria-labels vs dynamic titles
**Learning:** Dynamically changing the `aria-label` of a button based on its state is an accessibility anti-pattern. The accessible name of a control should identify its *purpose* (e.g., "Send message"), while its state (e.g., "disabled") should be conveyed via the `disabled` or `aria-disabled` attributes. Changing the `aria-label` to "Cannot send empty message" overrides the button's name, which can confuse screen reader users who use virtual cursors to explore the page. The dynamic `title` is sufficient for visual tooltips, but the `aria-label` should ideally remain static.
**Action:** When improving disabled states with explanations, apply dynamic explanatory text to the `title` attribute for visual hover tooltips, but ensure the `aria-label` (if used for an icon-only button) remains a static description of the button's purpose.
