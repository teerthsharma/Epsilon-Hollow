## 2024-06-01 - Dynamic Tooltips for Disabled States
**Learning:** Found that static tooltips like "Send message" on disabled submit buttons can be confusing, particularly when users don't know *why* the button is disabled. In React/Next.js, toggling the `title` attribute dynamically based on the same condition that disables the button (`!input.trim() ? "Enter a message to send" : "Send message"`) provides immediate contextual feedback for screen reader and mouse users alike.
**Action:** When creating form submit buttons, always tie the `title` or `aria-label` attribute dynamically to the validation state, and ensure visual disabled styling (`disabled:cursor-not-allowed`) is coupled with `focus-visible` styles for comprehensive accessibility.

## 2024-06-30 - Prevent Ghost Interactions During Offline States
**Learning:** Found that static interactive elements (like chat inputs and send buttons) lead to user confusion if the underlying backend connection or "tunnel" is offline. Users may attempt to interact with the system without realizing their actions are silently failing. Tying the disabled state of interactive elements directly to the system's connection status (e.g., `tunnelStatus !== 'LOCKED'`), along with dynamic visual cues (like an updated placeholder and title), provides immediate feedback that the system is unreachable.
**Action:** In components utilizing system hooks (like `useApeiron`), always link user input and button `disabled` states to the connection `tunnelStatus` and provide clear visual feedback to prevent ghost interactions during offline states.
