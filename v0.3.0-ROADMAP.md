## Current State (Brutal Honesty)

Seal OS v0.2.x boots to a graphical desktop. The kernel has REAL bare-metal primitives:
- ✅ Context switch with XSAVE/FXSAVE
- ✅ e1000 NIC driver (MMIO, descriptor rings, TX/RX)
- ✅ AHCI SATA driver (DMA, FIS, IDENTIFY)
- ✅ 4-level page tables, demand paging, SMP bringup
- ✅ Syscall entry/exit via `syscall`/`sysret`
- ✅ Security: SMAP/SMEP, seccomp BPF, MAC LSM, ASLR

**BUT:**
- ❌ The "emergency shell" is a **2-byte ELF** (`eb fe` = `jmp $`). It does nothing.
- ❌ `spawn_user` exists and works, but we feed it a screensaver instead of a program.
- ❌ TCP builds packets but **never calls the e1000 transmit function**.
- ❌ DHCP sends Discover, then **ignores the Offer**.
- ❌ WiFi returns `Simulated-AP-5G`. Bluetooth returns `[Sim] AirPods Pro`.
- ❌ Media player has zero decode logic. HTTP/TLS are stubs.
- ❌ `static mut AP_PER_CPU_PTR` triggers Rust 2024 warnings.
- ❌ README claims 15 codecs, full TLS, WiFi, Bluetooth — all fiction.

**Verdict: The engine runs. The car has no wheels.**

## v0.3.0 Goals (Achievable, Real, No Bullshit)

### 1. Real Userspace Init Program [P0 — BLOCKING]
**What:** Replace the 2-byte `jmp $` ELF with a real static ELF64 program.
**What it does:**
- `SYS_WRITE(fd=1, "Hello from Seal OS userspace!\n", 30)` → prints to serial
- `SYS_EXIT(0)` → exits cleanly

**Why this matters:** Without this, the OS is a kernel that boots to a desktop and then runs an infinite loop. A real userspace program proves the entire syscall→ELF→page table→context switch chain works end-to-end.

**How:**
1. Write a `#![no_std]` Rust program or hand-assemble a minimal ELF64.
2. Compile to a static position-independent executable.
3. Embed the bytes into `kernel/seal-os/src/process/userspace.rs` as `EMERGENCY_SHELL_ELF`.
4. Verify `spawn_user` loads it, `iretq` drops to ring 3, `syscall` returns to kernel, output appears on serial.

**Acceptance:** Boot in QEMU, see `[userspace] Hello from Seal OS userspace!` on serial.

---

### 2. TCP Actually Transmits Packets [P0 — BLOCKING]
**What:** Wire `tcp::send_tcp_packet()` to call `e1000::send_packet()`.
**What it does:** When TCP needs to send a SYN/ACK/data packet, it actually puts it on the wire.

**Why this matters:** Without this, the entire network stack is dead code. The e1000 driver is real. The TCP header builder is real. They just aren't connected.

**How:**
1. In `net/tcp.rs`, after constructing the TCP+IPv4 packet buffer, call `crate::drivers::net::e1000::send_packet(&buf)`.
2. Same for UDP, DHCP, DNS, ARP — ensure every protocol that builds a packet actually transmits it.
3. Handle the case where e1000 is not present (QEMU without `-device e1000`) — return honest error, no panic.

**Acceptance:** QEMU with `-device e1000` shows ARP, DHCP Discover, and eventually DHCP Ack in Wireshark or QEMU's network dump.

---

### 3. DHCP Completes Handshake [P1]
**What:** Implement DHCP Offer → Request → Ack state transitions.
**Current state:** `dhcp.rs` sends Discover, then does nothing with the response.

**How:**
1. Parse incoming DHCP packets in `net::poll()`.
2. On Offer (DHCPOFFER), extract offered IP, server IP, send Request.
3. On Ack (DHCPACK), set the interface IP, gateway, DNS.

**Acceptance:** `net::dhcp::client_ip()` returns a non-zero IP after boot.

### 4. Honesty Pass — Remove All Fake Data [P1]
**What:** Scrub every `[Sim]` prefix, `Simulated-`, and fabricated return value.

**Files to fix:**
- `drivers/wifi.rs` — return `Err("no WiFi controller detected")` instead of fake APs
- `drivers/bluetooth.rs` — return `Err("no Bluetooth controller detected")` instead of fake devices
- `apps/media_player.rs` — return `Err("no codec available")` instead of fabricated `MediaInfo`
- `net/http.rs` — return `Err("HTTP not implemented")` instead of simulated responses
- `net/tls.rs` — return `Err("TLS not implemented")` instead of stub responses
- `lang/mod.rs` — return `Err("Aether-Lang not integrated")` instead of `[Sim] pending...`
- `pkg/mod.rs` — return `Err("network unavailable")` instead of fake install success
- `async_rt/mod.rs` — document that it's a placeholder executor

**Acceptance:** `grep -r "\[Sim\]" kernel/seal-os/src/` returns zero matches. Boot log contains no fake device names.

---

### 5. Fix `static mut AP_PER_CPU_PTR` [P1]
**What:** Convert to `AtomicU64` or per-CPU safe storage.
**Current state:** `cpu/smp.rs` has `static mut AP_PER_CPU_PTR: u64 = 0;` used with `&mut *`.

**How:**
1. Replace with `static AP_PER_CPU_PTR: AtomicU64 = AtomicU64::new(0);`
2. Update all read/write sites to use `.load()`/`.store()`.

**Acceptance:** `cargo +nightly build --release` shows zero `static_mut_refs` warnings.

## Out of Scope for v0.3.0 (Be Realistic)

These are NOT v0.3.0 goals. They are v0.4.5+ or "maybe never":
- ❌ ManifoldFS persistence (needs block I/O layer + on-disk format = weeks)
- ❌ Aether-Lang interpreter completion (excluded from workspace; major project)
- ❌ Real WiFi/Bluetooth drivers (chipset-specific; months each)
- ❌ GPU acceleration (vendor drivers per GPU; months each)
- ❌ Media codec implementation (H.264, AAC, etc.; years of work)
- ❌ TLS/HTTPS (need crypto library first)
- ❌ Self-hosting HollowLang (does not exist)
