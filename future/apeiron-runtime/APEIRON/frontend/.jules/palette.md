## 2024-07-04 - Handling Offline Interaction States
**Learning:** Users can experience "ghost interactions" if inputs and buttons remain active while the underlying system (e.g., DSP Bus/Tunnel) is offline or disconnected. Providing clear disabled states with explanatory placeholders and ARIA labels prevents user confusion.
**Action:** Always link user input and button disabled states to connection status and provide clear visual feedback to prevent interactions during offline states.
